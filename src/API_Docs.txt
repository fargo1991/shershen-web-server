
API разеделно на несколько зон.

 Зона CUSTOMER
 -------------
  - Регистрация пользователей (покупателей/автовладельцев) системы. user.role = "CUSTOMER"

  Регистрация CUSTOMER-а:

      Происходит в несколько шагов. Сначала создается профиль пользователя. Затем, обязательно происходит привязка профиля к номеру телефона.

    POST /signup

    Content-Type: application/json

    {
        login : String,  // - length: min 5 max 30;
                            - Имя не должно состоять только из цифр;
                            - Может содержать следующие спец-символы: /_/./-/[/]/ Не может состоять из пробелов;
                            - Допускается любой регистр;
        password : String // - length: min 6, max 30;
                             - Пароль не может состоять только из цифр.
                             - Не должен совпадать из списка top 500 самых популярных паролей (? - нужно ли это..)
                             - не может содержать следующие спец-символы: /` \.
        phone : String // - проверка на валидность в соответствии с международным форматом
                            Длина строки не должна превышать 11 символов. Допускаются только цифры.
                          - Данное поле должно отобрадаться пользователю в соответствии с маской: +X (ХХХ) ХХХ-ХХХХ
                          (!) необязательное поле, если указан email
    }

    (!!) - Важно предупреждать пользователя при каждом действии в системе о необходимости привзяки аккаунта к номеру телефона.
        Пользователь не сможет бронировать, оставлять отзывы, совершать покупки и заказывать любые виды услуг,
        если аккаунт не привязан к номеру телефона.

    В случае успешного проходления запроса. Пользователю отправляется смс с кодом. Код должен быть указан в следующем запросе:


     POST /phone/confirm

     Content-Type : application/json



  - Поиск ближайших заведений(STORES) (работа со списком, фильтры по оказываемым услугам, ценам, расположению, рейтингу)
  - Работа с заведением (STORE) со стороны заказчика услуги. Фотографии, интро, выбор даты и времени записи в расписании,
  отзывы, построение маршрута на карте, оплата при подтверждении заявки со стороны STORE.

    (?) - Необходимо продумтаь способы оплаты услуги (3ds, привязка карты к приложению, внутренняя валюта).
    (?) - Желательно заложить в модуль оплаты потенциальную возможность оплаты купонами.
